Positioning with CSS

1/12/16

What We'll Learn

Does it seem like every time you try to position a node in CSS, it's down to trial and error? If someone were to ask you, "How did you manage to position that image like that?" would you look like this...

Positioning can be a tricky subject if you do not fully understand the attribute and how to it works between children and their ancestors. This blog post is a brief review of html positioning including the difference between static, relative, fixed, and absolute positioning. In addition to positioning, we will review the z-index property, what it does, and why you need it (Hint: It is very useful when elements start to overlap). There are also a few very handy resources I used to help understand this topic listed at the bottom of the page. If this post does not give you a truly solid understanding of positioning in CSS, please look at these resources to help complete your understanding!

What is Positioning

Positioning in CSS defines where you place an element on the page. It is a property assigned to an element. Check out the code below to see how a relative position would be assigned to main. 

            main {
              position: relative;
            }

There are four different assignments for positioning: Static, relative, absolute, and fixed.

Static

This is every element's position by default. You do not need to worry about defining an element's position as static, unless you are working to override a previous position command. 

            main {
              position: static;
            }

Relative

First, you need to remember that static is an element's default position. This is where an element would naturally fall without any outside adjustment. Relative positioning means placing the element relative to that static position. Relative to where the element would normally be placed. 

          main {
            position: relative;
            top: -20px;
            left: 10px;
          }

The code above places element main about -20 pixels up and 10 pixels to the left of where it would have been otherwise.

Fixed

A fixed element is positioned relative to the viewport. That means this element moves with the page as the user scrolls. Scroll up, scroll down, scroll all around. It doesn't matter to the example below. main will always be just 50 pixels from the top and 30 pixels from the right of your viewport. 

          main {
            position: fixed;
            top: 50px;
            right: 30px;
          }

Absolute

Absolute can be the trickiest position to understand. When an element is defined as absolute, it is positioned relative to the nearest positioned ancestor. If there is no such ancestor, the element is positioned relative to the page and will almost act as a fixed element, moving as the user scrolls. 

          main {
            position: absolute;
            top: 30px;
          }

Again, absolute says the element is placed relative to the nearest positioned ancestor. That means the parent, grandparent, etc. closest to the element that has a defined position (not static, but something like position:relative). For example, if an element of class #section2 was the child of #section1, in order for #section2 to be absolute relative to #section1, the code would have to look something like this: 

          #section1 {
            position: relative;
          }

          #section2 {
            position: absolute;
            top: 30px;
          }

If you are still unsure about absolute positioning, please check out some of the links at the end of this article.

Bonus: About the z-index

So where does the z-index come in? The z-index is relative when elements overlap. As you start positioning things around a page it is likely they will hide or cover each other. When your elements begin to overlap, you can use the z-index to assign an order of elements, also known as a stack order of elements. The z-index can be negative or positive!

Take the code below. In this case, the img would show up before the article, as it has a higher z-index, while the aside would be below everything else with a value of -1. 

          img {
            z-index: 2;
          }

          article {
            z-index: 1;
          }

          aside {
            z-index: -1;
          }

Now, imagine for a second that the z-index for these elements was never defined. What would the new order be? HTML automatically places elements at the bottom of the HTML code on top. The new order would be aside, article, img.
